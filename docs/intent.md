# Intent - 意图层

## 版本
v0.1.0

---

## 1. 为什么做这个项目？

### 背景与动机
<!-- 描述项目的起源、解决什么痛点 -->
- 需要一个简约、大气的个人博客平台
- 博客主要用于技术分享，也会有一些生活日常内容

### 期望价值
<!-- 项目成功后能带来什么价值 -->
- 建立个人技术品牌
- 积累和沉淀知识
- 与读者建立连接

---

## 2. 目标用户是谁？

### 主要用户：发布者（博客作者）
<!-- 描述核心用户画像 -->
- 角色：博客内容创作者
- 核心需求：便捷地发布和管理博客内容
- 使用场景：通过编写 Markdown 文件来创作内容

### 次要用户：读者
<!-- 描述次要用户画像 -->
- 角色：技术社区成员、普通访客
- 核心需求：获取有价值的技术内容和生活分享
- 使用场景：通过搜索引擎或直接访问博客阅读内容

---

## 3. 期望结果 / 验收标准

### 必须达成（Must Have）
- [x] 发布者能通过撰写 `.md` 文件发布内容
- [x] 读者访问首页即可获取最新内容
- [x] 高度 SEO 优化
- [x] 页面合理排版，导航、分类清晰

### 期望达成（Should Have）
- [x] 良好的阅读体验（代码高亮、目录导航等）
- [x] 响应式设计，移动端友好

### 可选达成（Nice to Have）
- [ ] 文章搜索功能
- [x] RSS 订阅

---

## 4. 明确不做什么

> 以下内容不在本项目范围内，避免范围蔓延。

- 不做用户登录/注册系统
- 不做评论系统（可考虑第三方集成）
- 不做后台管理界面（直接通过文件系统管理）
- 不做多语言支持（初期只支持中文）
